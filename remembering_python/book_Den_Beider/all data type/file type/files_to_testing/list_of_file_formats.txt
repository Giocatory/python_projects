Носитель для хранения видеоигр
Список наиболее распространенных расширений имен файлов, используемых при копировании образа ПЗУ игры или носителя информации с оригинального устройства с постоянной памятью (ROM) во внешнюю память, такую как жесткий диск, для резервного копирования или для воспроизведения игры с помощью эмулятора. В случае программного обеспечения на базе картриджа, если не используется расширение, зависящее от платформы, то расширения имени файла ".rom" или ".bin" обычно используются для уточнения того, что файл содержит копию содержимого ROM. Образы ПЗУ, дисков или магнитных лент обычно состоят не из одного файла или ПЗУ, а из целого файла или структуры ПЗУ, содержащихся в одном файле на носителе резервной копии.[33]

Общие форматы данных
Эти форматы файлов довольно четко определяются долгосрочным использованием или общим стандартом, но содержимое каждого файла часто очень специфично для конкретного программного обеспечения или было расширено дополнительными стандартами для конкретных целей.

Текстовый
CSV – значения, разделенные запятыми
HTML – язык разметки гипертекста
CSS – каскадные таблицы стилей
INI – текстовый файл конфигурации, формат которого в разных приложениях существенно схож
JSON – объектная нотация JavaScript - это открыто используемый формат данных, который в настоящее время используется многими языками, а не только JavaScript
TSV – значения, разделенные табуляцией
XML – формат открытых данных
YAML – формат открытых данных
reStructuredText – открытый текстовый формат для технических документов, используемый в основном на языке программирования Python
.md – Markdown открытый облегченный язык разметки для создания простого, но расширенного текста, часто используемый для форматирования файлов README
AsciiDoc – открытый формат документа разметки, доступный для чтения человеком, семантически эквивалентный DocBook
.yni – конфигурационный файл, аналогичный YAML
Общие расширения файлов
Это расширения файлов и широкие типы, часто используемые различными программами с разными форматами или без определенного формата.

Двоичные файлы
.bak, .bk – Файл Bak различных форматов резервного копирования: некоторые просто копии файлов данных, некоторые в форматах резервного копирования данных для конкретных приложений, некоторые форматы для общих программ резервного копирования файлов
BIN – двоичные данные, часто дампы памяти с исполняемым кодом или данными для повторного использования тем же программным обеспечением, которое их создало
DAT – файл данных, обычно двоичных данных, принадлежащих программе, которая его создала, или потоку Video CD в формате MPEG-1
DSK – файловые представления различных образов дисковых хранилищ
RAW – необработанные данные
SZH – файлы, которые ассоциированы с нулевыми уникальными типами файлов (наиболее распространенным является двоичный формат данных)
Текстовые файлы
.cnf, .conf, .cfg – конфигурационный файл, существенно зависящий от программного обеспечения
.log – файлы журнала обычно текстовые, но иногда двоичные
.asc, .txt – обычный текст, удобочитаемый человеком, обычно не более специфичный
Частичные файлы
Различия и исправления
diff – отличия текстовых файлов, созданные программой diff и применяемые в качестве обновлений с помощью патча
Неполные переводы
.!ut – !UT частично полная загрузка через uTorrent
.crdownload – загрузка CRDOWNLOAD частично полной или незаконченной из Google Chrome или Microsoft Edge
.opdownload – частичная или неполная загрузка OPDOWNLOAD из Opera
.часть – частично завершенная загрузка Mozilla Firefox или Transmission
.частичная – ЧАСТИЧНАЯ, частично полная загрузка Internet Explorer или Edge Legacy
Временные файлы
.temp, .tmp – временный файл, иногда в определенном формате, но часто просто необработанные данные в процессе обработки
Псевдопроводы, псевдопроволока - псевдопроволока файл, используемый для имитации программного канала
